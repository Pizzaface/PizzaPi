# Task: Refactor infrastructure & utilities

## Description

Set up the modular router infrastructure and extract shared utilities. This task is the foundation for all subsequent router extraction work. Update the central dispatcher in `handler.ts` to implement sequential path-based routing to domain-specific routers, and extract common logic into shared utility modules.

## Acceptance Criteria

- [ ] `handler.ts` implements router dispatcher pattern with sequential path matching
- [ ] All auth validation happens in `handler.ts` before router delegation

  - `parseJsonArray(input: unknown): unknown[]`

- [ ] No performance regression in dispatcher logic (measure baseline latency)

## Technical Details

### Architecture Changes

1. **Router Dispatcher Pattern**
   - Implement sequential path matching in `handler.ts`
   - Route patterns: `/api/auth/*` → auth router, `/api/runners/*` → runners router, etc.
   - Routers implement signature: `(req: Request, url: URL) => Promise<Response | undefined>`
   - Return `undefined` if path doesn't match; dispatcher tries next router
   - Central handler remains entry point; no new abstraction layers

2. **Auth Validation**
   - Preserve existing `requireSession` and `validateApiKey` checks in handler.ts
   - Auth middleware runs before router dispatch
   - Security posture unchanged

3. **Utility Extraction**
   - Move from current locations (likely embedded in api.ts or other files) to `routes/utils.ts`
   - All callers must be updated to import from new location
   - Utilities should have no external dependencies beyond Bun/Node standard library

4. **Type Safety**
   - Create optional `routes/types.ts` if shared request/response shapes exist
   - Ensure all types are properly exported and imported
   - No `any` types; strict mode compliance
