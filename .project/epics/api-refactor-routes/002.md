# Task: Extract auth & settings routers

## Description

Create the first two modular routers as foundational examples for the pattern. These routers have minimal external dependencies, making them ideal templates for the more complex routers that follow. Implement the auth router (user registration, signup status) and settings router (user preferences like hidden models).

### Testing Patterns

Routers should be testable without spinning up full server:

```typescript
// Example test pattern
import { handleAuthRoute } from "./auth";

test("POST /api/register returns 200 with token", async () => {
  const req = new Request("http://localhost/api/register", {
    method: "POST",
    body: JSON.stringify({ email: "user@example.com", password: "secret" })
  });
  const url = new URL(req.url);
  
  // Mock auth context (should be injectable or mockable)
  const res = await handleAuthRoute(req, url);
  
  expect(res?.status).toBe(200);
  const data = await res?.json();
  expect(data?.token).toBeDefined();
});
```
