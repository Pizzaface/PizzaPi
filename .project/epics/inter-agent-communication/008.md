# Task: Web UI Group Status Badges + Group Management Controls

## Description

Add group-awareness to the web UI. Session cards in the session list show a badge like "3/5 completed" for group coordinators (sessions that spawned children via `fan_out` or multiple `spawn_session` calls). The session viewer gets lightweight group management controls — view group members, see per-member status, and optionally cancel pending group members. Data is derived from child session heartbeats and channel membership.

## Acceptance Criteria

- [ ] **Group status badge** on session cards in the session list view
- [ ] Badge shows: `"{completed}/{total} completed"` for sessions with children
- [ ] Badge updates in real-time as child sessions complete
- [ ] Badge only appears for sessions that have at least one child session
- [ ] Badge styling: pill-shaped, muted background, consistent with existing badges
- [ ] **Group members panel** in session viewer (extension of topology tree from Task 007)
- [ ] Shows list of child sessions with individual status (active/idle/completed/error)
- [ ] Each child shows: session name, model, status indicator, duration
- [ ] Completed children show result summary (truncated)
- [ ] **Channel membership indicator** (if session is in any channels)
- [ ] Shows channel names the session belongs to
- [ ] Shows member count per channel
- [ ] Data source: aggregated from child session heartbeats + channel membership from server
- [ ] All existing UI tests pass
- [ ] Responsive design: badges and panels work on mobile

## Technical Details

### Data Sources

- **Child session state**: Derived from `childSessionIds` in session state (from Task 001) + individual child heartbeats
- **Completion counts**: Count children with `status === 'completed'` vs total children
- **Channel membership**: New `channel_membership` events from server (Task 005) forwarded to viewers

### Component Changes

**`packages/ui/src/components/session-card.tsx`** (or equivalent session list item):
- Add `GroupStatusBadge` sub-component
- Props: `childSessionIds: string[]`, `sessions: SessionInfo[]` (to look up child statuses)
- Compute: `completed = children.filter(c => c.status === 'completed').length`
- Render: `<Badge>{completed}/{total} completed</Badge>` (only if `total > 0`)

**`packages/ui/src/components/group-members-panel.tsx`** — new:
- Shows detailed list of child sessions
- Each row: name, model badge, status dot, time elapsed, truncated result (if completed)
- Integrates into session viewer sidebar below topology tree

**`packages/ui/src/components/channel-indicator.tsx`** — new:
- Small indicator showing channel memberships
- Shows: channel name + member count
- Data from `channel_membership` events

### Integration Points

- **Session list view**: Inject `GroupStatusBadge` into session card
- **Session viewer sidebar**: Add `GroupMembersPanel` below topology tree (Task 007)
- **Session state hooks**: Extend to track channel membership per session

### Styling

- Badge: `inline-flex items-center rounded-full px-2 py-0.5 text-xs bg-muted text-muted-foreground`
- Group panel: consistent with topology tree styling from Task 007
- Channel indicator: subtle, inline with session metadata
