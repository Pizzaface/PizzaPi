# Task: Protocol + Server Parent/Child Tracking and Completion Forwarding

## Description

Extend the protocol types and server relay to support parent/child session relationships and automatic completion forwarding. This is the foundation task — all other tasks in the epic depend on it. It adds `parentSessionId` tracking to Redis session data, extends protocol events for completion notifications, and implements server-side forwarding of completion results to parent sessions.

## Acceptance Criteria

- [ ] Protocol types in `packages/protocol/src/relay.ts` include `session_completion` event (client→server and server→client)
- [ ] `session_message` event type extended with optional `metadata` field for structured payloads (backward-compatible)
- [ ] `RedisSessionData` in `packages/server/src/ws/sio-registry.ts` includes `parentSessionId: string | null`
- [ ] `children:<sessionId>` Redis set maintained for fast child lookup
- [ ] `registerTuiSession()` accepts and stores `parentSessionId`
- [ ] Hub broadcasts (`session_added`, `session_removed`, heartbeats) include `parentSessionId` and `childSessionIds`
- [ ] Relay handler for `session_completion`: looks up parent socket, forwards as event to parent
- [ ] On `disconnect`: fires completion/error event to parent if session had a `parentSessionId`
- [ ] Spawn endpoint in `packages/server/src/routes/runners.ts` accepts `parentSessionId` in request body and passes it to registration
- [ ] All existing tests pass (`bun run test`)
- [ ] New unit tests for parent/child registration, completion forwarding, and disconnect cleanup
- [ ] `bun run typecheck` passes with no errors

## Technical Details

### Protocol Changes (`packages/protocol/src/relay.ts`)

- Add to `RelayClientToServerEvents`:
  ```typescript
  session_completion: (data: { sessionId: string; result: string; tokenUsage?: object; error?: string }) => void
  ```
- Add to `RelayServerToClientEvents`:
  ```typescript
  session_completion: (data: { sessionId: string; parentSessionId: string; result: string; tokenUsage?: object; error?: string }) => void
  ```
- Extend existing `session_message` event with optional `metadata?: Record<string, unknown>` field

### Server Changes (`packages/server/src/ws/`)

- **`sio-registry.ts`**: Add `parentSessionId` to `RedisSessionData`, update `registerTuiSession()` signature, manage `children:<sessionId>` Redis set (add on register, clean up on deregister)
- **`namespaces/relay.ts`**: Add `session_completion` handler — look up parent session's socket via registry, forward the completion event. Handle disconnect by sending error/completion to parent.
- **`sio-state.ts`**: Include `parentSessionId` and `childSessionIds` in session state broadcasts

### Route Changes (`packages/server/src/routes/runners.ts`)

- Extend spawn endpoint to accept `parentSessionId` in the request body
- Pass `parentSessionId` through to the runner's session creation flow
- Set `PIZZAPI_PARENT_SESSION_ID` environment variable for the spawned session

### Files Affected

- `packages/protocol/src/relay.ts`
- `packages/server/src/ws/sio-registry.ts`
- `packages/server/src/ws/sio-state.ts`
- `packages/server/src/ws/namespaces/relay.ts` (or the relevant namespace handler file)
- `packages/server/src/routes/runners.ts`
- New test files for the above changes
