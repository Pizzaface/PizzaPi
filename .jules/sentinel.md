## 2025-03-05 - IP Spoofing via Unvalidated X-Forwarded-For
**Vulnerability:** The application was extracting client IP addresses for rate limiting directly from `req.headers.get("x-forwarded-for")?.split(",")[0]` without verifying if the request originated from a trusted proxy.
**Learning:** This implementation allows an attacker to easily bypass IP-based rate limits (such as on the `/api/register` endpoint) by manually injecting a spoofed `X-Forwarded-For` header. The server blindly trusts the header, tracking the fake IP instead of the attacker's actual IP. The Fetch API Request object mapping from Node's `IncomingMessage` hid the `socket.remoteAddress` making it harder to access securely.
**Prevention:** Always require explicit opt-in configuration (e.g., `PIZZAPI_TRUST_PROXY=true`) before trusting proxy headers like `X-Forwarded-For`. Pass the actual `socket.remoteAddress` from the underlying server down into the request context (e.g. via an injected `X-Real-IP` header) and fallback to it when not behind a trusted proxy. Use a centralized `getClientIp()` function for all rate limiting and security events to enforce this consistently.
